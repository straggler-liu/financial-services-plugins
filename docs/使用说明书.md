## 1. 工具简介

本工具包含两个独立模块，可按需单独使用：

### 模块一：智能晨报/晚报系统（market-morning-brief）

基于 Claude 大模型，每天自动在开盘前推送 A股/港股/美股 深度分析到飞书，收盘后真正复盘验证建议准确性。

### 模块二：中国股市分析工具（chinese-markets + standalone-app）

A股/港股数据查询、股票筛选、DCF估值、财报分析等，支持 Claude 插件调用和独立 Web 应用两种方式。

## 3. 部署前必读：设备与环境要求

> 本章节非常重要，请在安装前仔细阅读。

### 3.1 两个模块对设备的要求差异

| 要求项 | 晨报/晚报系统 | 独立Web应用 |
|--------|-------------|-----------|
| **电脑是否要一直开机** | **是**（需持续运行定时任务） | 仅使用时开机 |
| **最低推荐方案** | 云服务器 / 家用NAS | 普通电脑/笔记本 |
| **普通电脑能否用** | 能，但电脑关机时不推送 | 能 |
| **操作系统** | Linux/Windows/macOS | Linux/Windows/macOS |
| **Python版本** | 3.12+ | 3.8+ |
| **磁盘空间** | 约 500MB | 约 500MB |
| **网络** | 持续网络连接（推送飞书需要） | 使用时有网络即可 |

### 3.2 晨报/晚报系统：设备选择指南

系统需要在固定时间点（09:00 / 15:30 / 21:00 CST）自动运行，这意味着**运行设备必须在这些时间点处于开机并联网状态**。

#### 方案一：云服务器（强烈推荐）

**优点：** 24小时运行，稳定可靠，不占用个人电脑

**推荐配置：**
- 阿里云 / 腾讯云 / 华为云 轻量应用服务器
- 最低规格：1核1GB（已足够）
- 费用：约 ¥30~60/月
- 系统：Ubuntu 22.04 LTS

```bash
# 云服务器上使用 Docker 部署（推荐）
docker-compose up -d   # 后台运行，重启自动恢复
```

#### 方案二：家用 NAS / 软路由

**优点：** 一次性投入，长期运行成本低

- Synology / QNAP NAS：支持 Docker，直接部署
- 软路由（OpenWrt + Docker）：技术门槛较高
- 树莓派：成本约 ¥200~400，稳定可靠

#### 方案三：普通电脑/笔记本（有限制）

**限制：** 电脑关机或休眠时不会推送报告

**适合场景：** 测试体验、手动触发、不需要完全自动化推送

**Windows 保持运行的方法：**
- 控制面板 → 电源选项 → 从不睡眠
- 适合台式机，笔记本长期开机发热需注意

**后台运行命令：**
```bash
# Windows（命令提示符）
start /B python src/main.py > cache/nohup.log 2>&1

# Mac/Linux
nohup python src/main.py > cache/nohup.log 2>&1 &
```

|---------|---------|------|

- **操作系统**：Windows 10/11、macOS 10.15+、Linux

### 4.2 Docker 安装（推荐，适合云服务器）


#### 第一步：安装 Python

## 5. 晨报/晚报系统使用

### 5.1 自动推送时间表

| 时间 (CST) | 触发 | 内容 |
|-----------|------|------|
| 09:00 周一至五 | 自动 | A股+港股 开盘前分析 |
| 15:30 周一至五 | 自动 | A股+港股 收盘复盘 |
| 21:00 周一至五 | 自动 | 美股 开盘前分析 |

### 5.2 手动触发

```bash
# Docker 用户
docker-compose exec market-brief python src/main.py --now premarket_asia
docker-compose exec market-brief python src/main.py --now postmarket_asia
docker-compose exec market-brief python src/main.py --now premarket_us

# 本地用户
python src/main.py --now premarket_asia
```

### 5.3 分析引擎说明

系统优先使用 **Claude 大模型**进行第一性原则分析：

- 真正理解新闻语义（能处理否定句、同义词、复杂句式）
- 推理完整的传导链条（非关键词匹配拼接）
- 收盘时真正对比早盘预测与实际走势
- 推理受影响个股并给出逻辑理由

未配置 `ANTHROPIC_API_KEY` 时，自动降级为规则引擎（关键词匹配），分析质量显著下降，收盘复盘功能基本失效。

### 5.4 飞书推送设置

1. 打开飞书，进入任意群聊
2. 点击右上角 "设置" → "群机器人" → "添加机器人" → "自定义机器人"
3. 设置机器人名称（如"市场晨报"），点击"添加"
4. 复制 Webhook URL，填入 `.env` 的 `FEISHU_WEBHOOK_URLS`

支持多个群同时推送（逗号分隔多个 URL）。

## 9. 常见问题

### Q1: 晨报系统不推送了，怎么排查？

**检查步骤：**
1. 确认运行设备处于开机状态（云服务器最可靠）
2. 查看日志：`docker-compose logs -f` 或 `cat cache/market_brief.log`
3. 测试飞书连接：`python src/main.py --test`
4. 手动触发验证：`python src/main.py --now premarket_asia`

### Q2: 电脑关机了，能补推吗？

不能自动补推。可手动触发：开机后运行 `python src/main.py --now premarket_asia`。

建议使用云服务器避免此问题。

### Q3: 不想用 Docker，可以直接用 Python 吗？

可以。`pip install -r requirements.txt && python src/main.py` 即可。但 Docker 方式在服务器重启后可自动恢复，更适合生产使用。

### Q4: 没有 Anthropic API Key 可以用晨报系统吗？

可以运行，但会降级为规则引擎（关键词匹配）：
- 市场摘要：仅拼接指数涨跌幅，无实质分析
- 操作建议：模板化套话，无逻辑推理
- 收盘复盘：无法真正对比早盘建议，功能基本失效
- 个股分析：完全缺失

强烈建议配置 API Key（每日费用约 ¥0.4~1.1，约 $0.06~0.15）。

### Q5: 数据抓取失败怎么办？

系统设计了多层降级：每个数据源失败会自动跳过，不影响整体分析。查看日志 `cache/market_brief.log` 了解详情。

国内服务器访问境外数据源（Yahoo Finance、Investing.com）可能需要代理，在 `.env` 中配置 `HTTP_PROXY`。

### Q6: 怎么设置多个飞书群？

在 `.env` 中用逗号分隔多个 Webhook URL：
```
FEISHU_WEBHOOK_URLS=https://open.feishu.cn/.../hook/xxx,https://open.feishu.cn/.../hook/yyy
```

### Q7: 美国夏令时/冬令时怎么处理？

修改 `.env` 中的 `US_PREMARKET_HOUR`，夏令时设 21，冬令时设 22，然后重启服务。

| 时期 | 美股开盘 CST | 建议设置 |
|------|-------------|---------|
| 夏令时（3月第2周 ~ 11月第1周） | 21:30 CST | `US_PREMARKET_HOUR=21` |
| 冬令时（其余时间） | 22:30 CST | `US_PREMARKET_HOUR=22` |

### Q8: 安装依赖时报错怎么办？

使用国内镜像源：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q9: 港股数据显示为空？

Yahoo Finance 在中国大陆可能需要网络代理。配置 `HTTP_PROXY` 或使用境外服务器部署。

### Q10: Claude 每次分析大概消耗多少费用？

使用 `claude-sonnet-4-6` 模型，每次分析约 3,000~6,000 input tokens + 1,000~2,000 output tokens，每次费用约 $0.02~0.05。每日3次推送约 $0.06~0.15（约 ¥0.4~1.1）。
